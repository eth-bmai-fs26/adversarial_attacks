# FGSM Adversarial Attack Visualization

## What This Is

An interactive lecture demo showing how FGSM (Fast Gradient Sign Method) fools neural networks with imperceptible perturbations. Built for university professors to use in-class on a projector. The concept was designed through an 11-round multi-agent discussion (see `viz-output/converged-concept.md` for the full spec).

## Architecture Overview

The app is a **beat-based presentation** (like slides) with a persistent ε slider. Five beats tell a story:

- **Beat 0** — Panda cold open (8s cinematic hook, non-interactive)
- **Beat 1** — "The Crime" (ε slider + MNIST digit + probability bars + gauge)
- **Beat 2a** — "The Ghost" (stained-glass sign map reveal — the screenshot moment)
- **Beat 2b** — "Equal vs. Proportional" (FGSM vs gradient split comparison)
- **Beat 3** — "The Implication" (standard vs robust model toggle)

Plus: image gallery (50 MNIST images), Lab Mode (TF.js live inference), and optional 3D mode (R3F loss surface).

**Everything in the core demo (Beats 0-3) is precomputed** — zero live neural network inference. Precomputed JSON files are generated by Python scripts in `precompute/`. TF.js is only used in Lab Mode.

## Tech Stack

- **Frontend:** React 18 + TypeScript + Vite + Tailwind CSS 3
- **Precomputation:** Python + PyTorch (LeNet-5 on MNIST)
- **3D (optional):** React Three Fiber + Three.js
- **Live inference (Lab Mode only):** TensorFlow.js
- **Fonts:** Syne (display), JetBrains Mono (numbers), DM Sans (body) — self-hosted via @fontsource

## Data Schema

The precomputed JSON (`standard_model.json`) has this structure per image:
```
pixels[784], true_class, adversarial_class,
loss_grad_sign[784], margin_gradient[784], grad_magnitude[784],
dead_pixel_mask[784], epsilon_values[100], logits_at_eps[100][10],
margin_at_eps[100], probs_at_eps[100][10], epsilon_star,
fgsm_margin_dot, raw_gradient_attack_logits[100][10], raw_gradient_flipped[100]
```

Full type definitions are in `src/types/index.ts`.

## Key Implementation Conventions

- **All canvas rendering at 2× for Retina displays** — `canvas.width = size * 2`, CSS `width = size`
- **MNIST upscaling uses nearest-neighbor** — crisp pixel blocks, no blur
- **The sign map is constant across all ε values** — it's computed at the clean image
- **Clipping is honest**: `x_adv = clip(x + ε·sign, 0, 1)`, background pixels barely affected
- **Logit interpolation** for smooth slider: linear interpolation between precomputed ε grid points
- **Runner-up hysteresis**: 5 percentage point threshold before changing the displayed runner-up class
- **Magnetic snap on ε slider**: ±0.01 zone around ε*, 300ms ease-out to exact ε*
- **Label shatter animation**: 12 CSS clip-path polygon fragments, 300ms total
- **TF.js and R3F are lazy-loaded** — not in the main bundle

## What NOT to Do

- Do NOT add live neural network inference to the core demo (Beats 0-3). Everything is precomputed.
- Do NOT use a component library (MUI, Chakra, etc.). All components are custom-built.
- Do NOT add equations to the visualization. The equation `x_adv = x + ε·sign(∇_x J)` is the professor's job on the blackboard.
- Do NOT use FXAA for the 3D mode — use SMAA for cleaner mesh edges.
- Do NOT import TF.js or Three.js at the top level. They must be lazy-loaded.
- Do NOT use gradient magnitude for FGSM tile sizes. FGSM tiles are ALWAYS uniform (15×15px). Only the gradient comparison (Beat 2b right half) uses variable-size tiles.
- Do NOT use `animation-delay` chains for the Beat 0 timed sequence. Use JS `setTimeout` so the sequence can be skipped/reset cleanly.
